# SparkVM | CloudX BGP 系列产品清退补偿公告

发布时间：2026/04/11

适用范围：CloudX BGP 系列受影响产品

---

因上游线路调整，**CloudX BGP 系列产品已进行清退**。现将本次受影响产品、补偿方案及处理流程公告如下。

## 一、受影响产品

以下 CloudX BGP 系列 SKU 属于本次清退范围：

* `CloudX-DualV4-BGP-V6-2000G`
* `CloudX-DualV4-BGP-V6-1000G`
* `CloudX-V6-BGP-V6-500G`

## 二、服务截止时间

受影响服务的使用时间统一计算至 **2026/04/11 00:00（UTC+8）**。

自该时间起，相关产品原有服务能力视为停止提供。

## 三、补偿方案

针对本次清退，**SparkVM** 提供以下三种处理方案，用户可根据自身需求选择其一。

### 方案一：更换为对应同规格 CloudX NP 系列产品

对于**未检测出异常检测行为**的用户，允许选择此方案。

可更换为以下对应 SKU：

* `CloudX-DualV4-NP-2000G`
* `CloudX-DualV4-NP-1000G`
* `CloudX-V6-NP-500G`

#### 更换原则

1. **仅支持同规格替换**，例如：

   * `2000G` → `2000G`
   * `1000G` → `1000G`
   * `500G` → `500G`

2. 选择更换产品方案的用户，**无需重装系统**。

3. 原服务中的 **eth0 / eth2 IP 地址不会发生变化**。

4. 原 **SZX-BGP 线路对应网卡将被移除**，相关网络能力将不再保留。

5. 由于 **CloudX NP 系列产品价格低于 CloudX BGP 系列产品**，**SparkVM** 将通过**延长服务时长**的方式进行差额补偿。

6. 如原服务存在**循环折扣**，则在核算目标 CloudX NP 同规格产品的当前日均价格时，**将按该循环折扣后的实际价格计算**，而非按未折扣原价计算。

7. 选择更换产品方案后，**原服务已生效的循环折扣将继续保留并延续适用**于更换后的对应 CloudX NP 同规格产品。

#### 延长时长计算原则

按原服务剩余未使用价值，折算为目标 CloudX NP 同规格产品的可用时长。

可按以下公式理解：

$$
\text{新服务可用时长} =
\frac{\text{原服务剩余未使用金额}}
{\text{对应 CloudX NP 同规格产品当前日均价格}}
$$

如原服务存在循环折扣，则其日均价格按折后价格计算，即：

$$
\text{当前日均价格} =
\frac{\text{CloudX NP 同规格产品当前计费周期价格} \times \text{循环折扣系数}}
{\text{当前计费周期天数}}
$$


### 方案二：按未使用时长等比例退款

如不选择更换产品，**SparkVM** 将按服务**剩余未使用时间等比例退款**，并通过**原支付渠道原路退回**。

提交退款申请后，**财务预计将在 3 个工作日内完成处理**。

### 方案三：更换为同规格 CloudX-CT / CloudX-CU 系列产品

对于**未检测出异常检测行为**的用户，允许选择此方案。

#### 更换原则

1. **仅支持同规格替换**，例如：

   * `2000G` → `2000G`
   * `1000G` → `1000G`
   * `500G` → `500G`

2. 选择更换产品方案的用户，**无需重装系统**。

3. 原服务中的 **eth0 / eth2 IP 地址不会发生变化**。

4. 原 **SZX-BGP 线路对应 IP 将替换为 `DGM-CT` / `SZX-CU`**。

5. 由于 **CloudX-CT / CloudX-CU 系列产品价格低于 CloudX BGP 系列产品**，**SparkVM** 将通过**延长服务时长**的方式进行差额补偿。

6. 如原服务存在**循环折扣**，则在核算目标 CloudX-CT / CloudX-CU 同规格产品的当前日均价格时，**将按该循环折扣后的实际价格计算**，而非按未折扣原价计算。

7. 选择更换产品方案后，**原服务已生效的循环折扣将继续保留并延续适用**于更换后的对应 CloudX-CT / CloudX-CU 同规格产品。

8. 选择此方案的用户，**预计需要等待 3 天后才开始处理**。

#### 延长时长计算原则

按原服务剩余未使用价值，折算为目标 CloudX-CT / CloudX-CU 同规格产品的可用时长。

可按以下公式理解：

$$
\text{新服务可用时长} =
\frac{\text{原服务剩余未使用金额}}
{\text{对应 CloudX-CT / CloudX-CU 同规格产品当前日均价格}}
$$

如原服务存在循环折扣，则其日均价格按折后价格计算，即：

$$
\text{当前日均价格} =
\frac{\text{CloudX-CT / CloudX-CU 同规格产品当前计费周期价格} \times \text{循环折扣系数}}
{\text{当前计费周期天数}}
$$
